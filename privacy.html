1 Summary

✅  OrionPulseNet does not collect, transmit, or share any personal data. All network analysis runs entirely on your device. This policy explains exactly what the app does and does not do with your information.

We built OrionPulseNet on a simple principle: your network data belongs to you. The app measures, analyses, and displays your network metrics locally — nothing is sent to us or any third party by default.

2 Data we do not collect

OrionPulseNet does not collect any of the following:

✕
Personal information of any kind
✕
Network traffic content or payloads
✕
IP addresses or MAC addresses
✕
Device identifiers or fingerprints
✕
Usage analytics or crash reports
✕
Location data (beyond SSID — see below)
✕
Contacts, calendar, or files
✕
Browsing history or app usage

3 Location services

OrionPulseNet requests Location permission for one purpose only: to read your Wi-Fi network name (SSID) via the system wireless interface API. This is a platform requirement — the operating system classifies SSIDs as location-sensitive data.

  • Your physical location is never recorded, stored, or transmitted
  • Location data is never used for any purpose other than reading the SSID
  • The app functions fully without this permission — only the SSID display in the Device tab is affected
  • You can revoke this permission at any time via System Settings → Privacy & Security → Location Services

📍  We do not know where you are. We do not want to know. Location permission exists solely because the operating system requires it to read a Wi-Fi network name.

4 Network monitoring

All network monitoring is performed locally using system APIs on your device. No network traffic content is ever inspected, logged, or transmitted.

  • Bandwidth measurements are derived from interface byte counter deltas — not packet inspection
  • Latency measurements use TCP timing probes to a public IP address (1.1.1.1) — no user data is transmitted
  • DNS timing is measured by resolving publicly known domain names locally — results are displayed only on your device
  • CPU and memory are read via kernel statistics APIs — no process data is recorded or transmitted

5 On-device AI

OrionPulseNet uses an on-device AI model as its primary analysis engine. All AI processing occurs entirely on your device.

  • No prompts, network metrics, or AI responses are transmitted to any external server
  • AI insights, deep analysis, and copilot responses are generated and stored locally only
  • If the on-device model is unavailable, the app falls back to a deterministic rule-based engine — also entirely local

🧠  Your conversations with the AI Copilot never leave your device. By default, OrionPulseNet has no connection to any AI cloud service.

6 Optional Groq API (cloud AI)

OrionPulseNet offers an optional integration with the Groq API for cloud-powered AI analysis. This feature is:

  • Disabled by default — it is never active unless you explicitly enable it
  • User-configured — you must manually enter your own Groq API key in the AI Insights tab settings
  • Throttled — a 30-second cooldown is applied between cloud analysis requests
  • Transparent about what is sent — only network metric summaries (speeds, latency values, scores) are sent to Groq's API. No personal data, no IP addresses, no identity information is included
  • Locally stored key — your API key is stored in local app preferences only and is never transmitted to us

If you enable this feature, Groq's privacy policy applies to any data processed by their service.

7 Data storage

OrionPulseNet stores the following data locally on your device only — nothing is backed up to any cloud service by this app:

  • Network history — JSON file retained for up to 7 days, stored in the app's Application Support folder
  • Recent bandwidth readings and latency samples — held in memory only, cleared when the app closes
  • Anomaly log — last 100 anomalies held in memory only
  • Optional Groq API key — stored in local app preferences if you choose to configure it

8 Third-party services

OrionPulseNet contacts one external service during normal operation:

  • ipwho.is — contacted during Connection Intelligence scans to look up the geographic country of publicly resolved domain IP addresses. Only the IP addresses of well-known public domains (e.g. google.com, github.com) are submitted. No personal data, no your device's IP address, and no user information of any kind is included in these requests.

No other external services are contacted during normal operation. The optional Groq API (Section 6) is only contacted if you explicitly configure it.

9 Children's privacy

OrionPulseNet is rated 4+ and is not directed at children under 13. Because we collect no personal data from any user, there is no special handling required for children. We do not knowingly collect any information from minors.

We will not retroactively change how previously collected data is used, because we collect none.

Continued use of OrionPulseNet after a policy update constitutes acceptance of the revised terms.
