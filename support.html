How can we help?

Answers to the most common questions about OrionPulseNet — setup, AI features, metrics, and more.

Getting started
Does OrionPulseNet require an account or subscription? ▼

No. The app launches immediately with no account, login, or subscription required. All features are free and available from the moment you open the app.

Does it work on Wi-Fi and Ethernet? ▼

Yes. OrionPulseNet automatically detects your active network interface and monitors whichever connection is active — Wi-Fi or Ethernet. The Device tab shows your interface type and diagnostics for the active connection.

What system version is required? ▼

OrionPulseNet requires macOS 26.4 or later. On-device AI features require a compatible device with Apple Neural Engine support.

Wi-Fi & SSID
My Wi-Fi network name is not showing. How do I fix it? ▼

Displaying your Wi-Fi network name (SSID) requires Location permission. This is a system-level requirement — the SSID is treated as location-sensitive data by the operating system.

To enable it:

  • Open System Settings → Privacy & Security → Location Services
  • Find OrionPulseNet and set it to While Using
  • Return to the app and tap Scan in the Device tab
All other features (bandwidth, latency, AI insights, history) work fully without this permission. Only the SSID display is affected.
Why does a network monitoring app need Location permission? ▼

This is a platform requirement, not our choice. The operating system classifies Wi-Fi network names (SSIDs) as location-sensitive because they can be used to infer physical location. Any app that reads the SSID must have Location permission. Your actual GPS location is never accessed, recorded, or transmitted by OrionPulseNet.

Does OrionPulseNet track my location? ▼

No. Location permission is used solely to read your Wi-Fi network name. Your location is never recorded, stored, or transmitted anywhere. This is clearly stated in our Privacy Policy.

OrionPulseNet collects zero personal data of any kind.
AI features
What does "On-Device AI Ready" mean? ▼

"On-Device AI Ready" means the on-device AI model is available on your system and all AI analysis runs entirely locally on your device. Nothing is sent to any server. This is the default and preferred mode.

What does "Local Analysis Mode" mean? ▼

If the on-device AI model is not available on your device, OrionPulseNet automatically switches to a rule-based local analysis engine. You still receive full insights and recommendations — they are generated by a deterministic rule system rather than the AI model. All other features (bandwidth, latency, history, DNS, device diagnostics) work identically.

How do I use the optional Groq API for cloud AI? ▼

In the AI Insights tab, tap the Settings icon and enter your Groq API key. This enables cloud-powered analysis as a fallback or supplement. Your key is stored locally only and is never shared with anyone.

The Groq API is entirely optional and disabled by default. On-device AI is always the primary engine.
Does AI data leave my device? ▼

No — by default. When using on-device AI (the default), all processing is entirely local. If you optionally enable the Groq API, only network metric summaries (speeds, scores, latency values) are sent — no personal data, no IP addresses, no identity information.

How metrics are measured
How is download and upload speed measured? ▼

Speed is measured by reading network interface byte counters directly from the system. The difference between consecutive readings (every second) gives bytes transferred, converted to Mbps. Exponential moving average smoothing is applied to reduce noise. No speed test servers are used — this reflects your actual ongoing network usage.

How is latency measured? ▼

Latency is measured by opening a TCP connection to 1.1.1.1:443 and timing the round trip. A warmup probe is performed first to eliminate connection setup overhead, followed by 5 test probes. The average is reported. No data is transmitted — this is a pure timing measurement.

How is jitter measured? ▼

Jitter is calculated using the RFC 3550 method: the mean absolute difference between consecutive latency samples. This is the same standard used by VoIP quality measurement systems. Lower jitter means more consistent, predictable latency.

How is the Network Stability Score calculated? ▼

The score starts at 100 and deductions are applied for:

  • High latency — up to -30 points
  • High jitter — up to -25 points
  • Packet loss — up to -30 points
  • Bandwidth instability — up to -7.5 points

Grades: Excellent (90+), Good (75+), Fair (50+), Poor (25+), Critical (<25).

What triggers an anomaly? ▼

Anomalies are flagged when measurements exceed these thresholds:

  • Bandwidth spike more than 3x the recent average
  • Latency exceeds 100ms
  • Jitter exceeds 20ms
  • Packet loss exceeds 2%
  • DNS resolution exceeds 50ms
Bandwidth spike anomalies are common on healthy networks during large downloads or streaming. Use AI Insights for a contextual interpretation.
How much history is stored and where? ▼

Network history is retained for 7 days and stored locally as a JSON file in the app's Application Support folder on your device. It is never uploaded anywhere. You can clear it by deleting the app's Application Support folder, or it auto-clears after 7 days.
